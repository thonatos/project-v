## Context

docs-app 是一个基于 React Router v7 的文档展示应用，文档以 Markdown 文件存储在 `app/docs/` 目录。当前文档 frontmatter 包含 `title`、`date`、`description` 三个字段，通过 `app/lib/docs.ts` 进行解析和处理。

应用当前有两个路由：
- `/` - 文档列表页面，展示所有文档
- `/docs/:slug` - 文档详情页面，展示单个文档内容

本次设计为文档系统添加标签（tags）功能，不涉及外部依赖，不改变现有架构模式。

## Goals / Non-Goals

**Goals:**

- 支持文档 frontmatter 添加 `tags` 字段（数组格式）
- 提供标签列表页面展示所有标签及文档数量
- 提供标签详情页面展示该标签下的文档列表
- 在文档列表和详情页显示标签，支持点击导航

**Non-Goals:**

- 不实现标签管理功能（创建、编辑、删除标签）
- 不实现标签层级或父子关系
- 不实现标签搜索功能
- 不修改现有文档内容或样式

## Decisions

### D1: Tags 字段格式

**决定**: 使用 YAML 数组格式 `tags: [tag1, tag2]`

**理由**:
- YAML 数组格式简洁，与现有 frontmatter 风格一致
- 支持多标签，符合用户需求
- 可选字段，不破坏现有文档

**备选方案**:
- 字符串格式 `tags: tag1, tag2` - 需额外解析，增加复杂度
- 不支持多标签 - 功能受限

### D2: 标签路由设计

**决定**: 使用 `/tags` 和 `/tags/:tag` 路径

**理由**:
- 符合 RESTful 风格
- 与现有 `/docs/:slug` 路径结构一致
- URL 可读性强，便于用户理解

### D3: 标签数据获取

**决定**: 在 `docs.ts` 中新增 `getAllTags()` 和 `getDocsByTag(tag)` 函数

**理由**:
- 与现有 `getAllDocs()` 和 `getDocBySlug()` 函数模式一致
- 保持数据层职责单一
- 避免重复解析 frontmatter

### D4: 标签 UI 组件

**决定**: 创建 `TagBadge` 可复用组件，使用小徽章样式

**理由**:
- 组件化设计，便于多处复用（列表页、详情页、标签页）
- 徽章样式符合标签的视觉语义
- 点击跳转通过 `<Link>` 实现，保持 React Router 模式

## Risks / Trade-offs

**[R1] 标签大小写不一致** → 统一转换为小写存储和展示，URL 使用小写

**[R2] 空标签或无效标签** → 前端过滤空标签，忽略无效值

**[R3] 大量标签导致页面过长** → 暂不处理，后续可考虑分页或折叠

## Migration Plan

本次为纯新增功能，无需数据迁移：

1. 部署代码更新
2. 用户可在文档 frontmatter 中添加 `tags` 字段
3. 标签页面自动生效

**回滚**: 删除相关代码即可，不影响现有文档
