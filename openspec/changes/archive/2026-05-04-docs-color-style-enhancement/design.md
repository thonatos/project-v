## Goals / Non-Goals

**Goals:**
- 建立新的青绿色系配色方案
- 导航栏采用左右分栏布局，右侧链接用竖线分割，便于扩展
- 添加背景渐变色提升美观性
- 通过 card/线框增加站点层次感（小范围、克制使用）
- 优化标题、标签、链接的视觉表现
- 保持简约简洁的设计原则

**Non-Goals:**
- 不改变文档内容的结构
- 不增加复杂的交互功能
- 不引入新的外部依赖
- 不大范围使用 card/阴影效果（保持简洁）

## Decisions

### 1. 配色方案选择

**决策**：采用青绿色 (teal) 作为主色调，配合白色和灰色

**理由**：
- 青绿色充满活力又不失专业感
- 与白色背景形成良好对比
- 参考 Longbridge 的成功案例

**配色定义**：
- 主色：`#0d9488` (teal-600)
- 主色浅：`#14b8a6` (teal-500)
- 主色深：`#0f766e` (teal-700)
- 背景：`#ffffff`
- 文字：`#1f2937` (gray-800)
- 文字浅：`#6b7280` (gray-500)

### 2. 导航栏布局设计

**决策**：采用左右分栏布局，右侧链接用竖线分割

**布局结构**：
```
[Logo ρV undefined project] | [Tags] | [Tech] | [Trading] | [GitHub]
         左侧                  右侧链接区域（竖线分割）
```

**理由**：
- 参考 Longbridge Developers 导航栏设计
- 左侧 logo 区域固定，便于识别
- 右侧链接用浅灰色竖线 (`border-left`) 分割，便于未来扩展
- 响应式：桌面端显示全部，移动端隐藏部分链接

**样式实现**：
- 导航链接之间使用 `border-left: 1px solid gray-200` 分割
- 第一个链接不显示分割线
- hover 状态使用青绿色高亮

### 3. 背景渐变色

**决策**：添加微妙的渐变背景，从白色过渡到浅青绿色

**理由**：
- 提升视觉美观性
- 保持简约，不使用强烈渐变
- 与主色调协调

**样式实现**：
```css
body {
  background: linear-gradient(180deg, #ffffff 0%, #f0fdfa 100%);
}
```
- 渐变从纯白到 teal-50 (`#f0fdfa`)
- 角度 180deg（从上到下）

### 4. 层次感设计（克制使用）

**决策**：在关键位置使用 card/线框增加层次感

**使用位置**（克制、小范围）：
- 文档详情页的标题区域：使用浅色线框或背景
- Article Card：使用微妙的底部边框（已存在，保持）
- 代码块：保持现有样式

**不使用位置**：
- 不给所有内容区域添加 card
- 不使用明显阴影
- 不使用圆角卡片包裹大块内容

**样式实现**：
- 标题区域：`bg-gray-50/50 border border-gray-100 rounded-lg`
- 保持整体简洁风格

## Risks / Trade-offs

**风险**：配色变化可能影响用户习惯 → 提供回滚方案，保留原 CSS 变量注释

**风险**：渐变背景可能影响可读性 → 使用微妙渐变，确保文字对比度足够

**风险**：card 使用过多导致视觉复杂 → 明确限制使用范围，只在标题区域使用

**权衡**：更鲜艳的颜色 vs 保持简约 → 使用中等饱和度的青绿色，避免过度刺激

**权衡**：层次感 vs 简洁 → 克制使用 card，仅在小范围、关键位置使用

**迁移计划**：
1. 更新 CSS 变量定义和背景渐变
2. 重构 header 组件为左右分栏布局，添加竖线分割
3. 创建共享配色配置文件
4. 更新 tag-badge 和 tags._index 使用共享配置
5. 添加标题区域的线框样式（克制使用）
6. 新增 tech/trading 路由页面
7. 测试各页面视觉效果
8. 如需回滚：恢复原 CSS 变量和组件代码
